# Power BI 助手 — 使用说明

> 版本 1.1.0 ｜ AI 辅助编写 Power BI 的 **DAX 度量值**与 **Power Query（M）清洗脚本**。
> 所有生成都基于你**正在打开的** Power BI Desktop 真实模型（不臆造表/列），DAX 还会连引擎**实跑验证**、可一键写回。

---

## 一、它能做什么

| 助手 | 作用 | 验证方式 |
|------|------|----------|
| **DAX 助手** | 用自然语言生成度量值；或用「已知正确值」校准出准确口径 | 连本地引擎 **实跑 `EVALUATE`**，错了自动修，可一键写回模型 |
| **Power Query（PQ）助手** | 在某个查询当前已应用的 M 之上，补出清洗/转换步骤 | **静态校验**（结构 + 列引用 grounding）；由你粘进 Desktop 应用后真正验证 |

两个核心原则：
- **语境为王**：只引用你模型里**真实存在**的表、列、度量值、查询，杜绝编造。
- **真验证而非只建议**：DAX 走「生成 → 静态校验 → 实跑 → 修复」闭环；PQ 走「生成 → 静态 lint → 你在 Desktop 应用验证」。

---

## 二、运行前提

1. **Windows 电脑**（不支持 Mac）。
2. **已安装 Power BI Desktop**，并且**已打开要分析的报表**——本工具靠它读取模型、实跑验证。
3. 一个**大模型 API Key**（如通义千问 / DashScope），**每人用自己的**。

> ⚠️ **重要：每人在自己电脑上安装运行。**
> 本工具靠**本机**进程连接**本机**的 Power BI Desktop。它在哪台机器上运行，就只能连那台机器的 Power BI。
> 所以**不要**把你的网络地址（如 `http://10.x.x.x:8501`）发给同事当共享入口——那样他们连到的还是**你**的 Power BI。
> 正确做法：同事各自安装本工具，在**自己**的 `localhost:8501` 打开，连自己的报表。

---

## 三、安装与启动

推荐 **安装包版（exe）**，体验最干净：

1. 运行 `PowerBI助手_安装.exe`（免管理员、装到当前用户目录）。
2. 从「开始菜单 → Power BI 助手」启动，会自动弹出应用窗口（无命令行黑窗）。
3. **关闭应用窗口 = 关闭应用**，后台服务自动停止。

绿色版（zip）替代方案：

1. **解压到短路径**目录，例如直接放 `C:\PowerBI助手`。
   > Windows 有 260 字符路径上限，解压到很深/很长的目录（桌面套多层、OneDrive 深层）会导致部分文件释放失败。放 `C:\` 下最稳。
2. 双击 `packaging\启动.bat`，会先出现命令行窗口（正常），稍后自动弹出应用窗口。首次启动若在准备环境，请联网等待几分钟（仅首次）。
   > **请勿直接双击 `启动.vbs`**——很多电脑把 `.vbs` 默认用记事本打开（只弹代码不运行）。要无黑窗请用安装包。

## 四、首次配置

1. 点左上角 **⚙ 设置齿轮**，选择大模型供应商、填入**你自己的 API Key**，保存。
   - Key 仅保存在你本机（`C:\Users\你的用户名\.powerbi_ai_assistant\`），不会上传。
2. 左侧「**模型来源**」处的圆点是连接状态红绿灯：**红 = 未连接，绿 = 已连接**。
3. 点 **🔌 连接 Power BI Desktop**，变绿灯即连上当前打开的报表。
   - 若同时开了多个报表，会让你选择实例（按端口）。

---

## 五、界面总览

- **顶部主菜单**：`Σ DAX` / `🧹 PQ` 两个按钮，切换当前助手。
- **左侧栏随助手切换**：
  - 选 **DAX** → 显示「Power Pivot · 模型」（表与列、度量值），**点字段名可插入到需求框**。
  - 选 **PQ** → 显示「Power Query · 查询」（按文件夹分组的已加载查询），点查询展开列、点列名插入需求框。
- **两个开关**（DAX 两种模式下都有，样式一致）：
  - **✍️ 写入 Power BI**：开启后，验证通过的度量值可一键写回模型。
  - **🧠 深度思考**：让模型作答前先内部推理，更准但更慢。

## 六、DAX 助手

顶部用单选切换两种模式：**💬 基础 DAX 生成** / **🎯 校准式生成**。

### 1. 基础 DAX 生成
- 在底部输入框用自然语言描述想要的度量值，**回车**或点 ➤ 发送。
- 左侧点字段名（如 `'Sales'[Amount]`）可把真实引用插进输入框，减少口径歧义。
- 生成的每条度量值会**实跑 `EVALUATE` 验证**：通过显示绿色结果；引擎报错会显示并自动尝试修复。
- 开「✍️ 写入 Power BI」后，验证通过的度量值/计算表可一键写回（见第八节保存）。

### 2. 校准式生成（推荐用于口径易错的指标）
用「已知正确值」逼出准确口径——适合"看起来对、其实算错"的度量值。

1. 填 **业务需求**（如"本年度已审批的采购总额"）。
2. 搭 **校准点**：选 表/列/值 → **➕ 添加筛选条件** 组成一个切片 → 在 **该切片下的正确值** 填你手算确认过的数 → **➕ 添加为校准点**。
3. 可加**多个校准点**（**多点校准**：要求生成的度量值在**每个**切片都算对，更能锁死正确口径）。
4. 选 **匹配规则**（容差，默认"相对 0.01%"）。
5. 点 **开始校准**。AI 会生成 → 在每个切片实跑 → 对不上就自动修；实在是需求有歧义时，会**反问你一个关键问题**（如"是否含税？"）。

**填错了标准值？直接在对话里说就行。**
在校准对话框里输入「**第2个切片应该是 34.28**」「把切片2改成 100」之类，系统会：
- 自动识别这是**更正标准值**（而非澄清口径），
- **确定性地**改掉对应校准点的期望值，并回显一条 `✏️ 已按你的更正调整校准目标…` 让你核对，
- 用当前候选度量值**重新验证**——很多时候会立刻命中。

> 校准模式下「🧠 深度思考」默认**开启**（这里最值得用），可临时关掉求快。
> 命中后开「✍️ 写入 Power BI」即可一键写回该度量值。

## 七、Power Query（PQ）助手

1. 顶部选 **要清洗的查询**（只列出已加载进模型的查询）。
2. 用自然语言描述要补的清洗/转换步骤，发送。AI 会在该查询**当前已应用的 M** 之上续写。
3. 生成的 M 会做**静态校验**（结构、列引用 grounding）。
4. **应用方式**：复制代码框（右上角按钮）→ 在 Desktop 里 `主页 → 转换数据` → 选中该查询 → `高级编辑器` → 全选粘贴 → `完成` → `关闭并应用`。**"关闭并应用"时才真正验证。**
5. 你在 Desktop 改完查询后，点助手里的 **🔄 同步** 重新读取最新 M（助手不会自动刷新）。

> ⚠️ **PQ 助手是只读的、不直接写回**：通过外部连接往**已打开**的 Desktop 写 M 会导致 Desktop 的查询导航器崩溃。所以这里只给你验证过结构的 M，由你粘贴应用——这是有意为之的安全设计。

---

## 八、保存到报表

- 一键写回（DAX 度量值/计算表）是写进**当前打开的模型内存**。
- **写回后必须在 Power BI Desktop 里按 `Ctrl+S`**，改动才会真正落盘保存。

---

## 九、常见问题

- **同事打开只能连到我的 Power BI**：见第二节——本工具是本机运行、连本机 Power BI。请让同事各自安装、在自己电脑运行。
- **浏览器打开是错误页/连不上**：稍等几秒刷新，服务还在启动。
- **左侧连不上 Power BI**：确认 Desktop 已打开报表；切换报表后点一下「连接」重连（端口每次重开会变，工具会自动找）。
- **第一次写回度量值较慢/报错**：首次写回需联网下载一个 ~30MB 的组件（仅一次）；公司网络受限时可能需要稍候或重试。
- **AI 回复慢**：把「🧠 深度思考」关掉（基础生成默认就是关的），简单问题会快很多。
- **改了应用看不到变化**：刷新浏览器（Ctrl+F5）。
