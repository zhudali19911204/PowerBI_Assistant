# 打包说明（给分发者）

目标：产出一个压缩包，内部同事**解压、双击 `packaging\启动.bat` 即用**，唯一前提是他们本机装了 Power BI Desktop。

应用必须**在每台机器本地运行**（它连的是本机 `localhost` 上 Power BI Desktop 的引擎端口），所以**不能挂服务器统一访问**，只能分发本地包。

---

## 方案一：零前提便携包（推荐，同事无需装 Python）

思路：把一个**可重定位的便携 Python**连同依赖一起打进包里，放到 `runtime\`。`启动.bat` 检测到 `runtime\python.exe` 就直接用它，同事什么都不用装。

### 步骤

1. **取一个可重定位的 Python 3.12（Windows x64）**
   从 `astral-sh/python-build-standalone` 的 Releases 下载形如：
   `cpython-3.12.*-x86_64-pc-windows-msvc-install_only.tar.gz`
   解压后会得到一个 `python\` 文件夹（内含 `python.exe`、`Scripts\`、`Lib\` 等），它是自包含、可整体拷贝的。

2. **把依赖装进这个便携 Python**（在本机联网执行一次）：
   ```bash
   # 假设解压出的目录是 ./python
   ./python/python.exe -m pip install --upgrade pip
   ./python/python.exe -m pip install -r packaging/requirements-runtime.txt
   ```

3. **组装分发目录**，结构如下：
   ```
   PowerBI助手\
   ├─ runtime\                      ← 上一步那个 python 文件夹，重命名为 runtime
   ├─ powerbi_ai_assistant\         ← 整个源码目录（从本仓库拷）
   ├─ .streamlit\                   ← 含 config.toml（锁定亮色主题）
   ├─ packaging\
   │   ├─ 启动.vbs               ← 双击入口（应用模式窗口，隐藏黑窗）
   │   ├─ 启动.bat               ← 工作脚本（也可直接双击作兜底）
   │   ├─ requirements-runtime.txt
   │   └─ 使用说明.md
   └─ 使用说明.md                    ← 可把 packaging\使用说明.md 复制一份到根目录方便看
   ```
   > 不需要拷 `.venv\`、`tests\`、`.git\`、`__pycache__\`、`docs\`、`.claude\`。

4. **（可选）预置写回组件，免首次联网下载**
   写回度量值用的 TOM 组件默认在"首次写回"时从 NuGet 下到每个用户的 `~/.powerbi_ai_assistant\lib\amo`。
   若同事网络受限，建议提前在本机跑一次让它下好，并告知同事首次写回需联网一次（读/生成/验证不需要）。

5. **打成 zip**：把 `PowerBI助手\` 整个压缩。发给同事即可。

> ⚠️ **路径长度（MAX_PATH）**：组装、打包、以及同事解压都**务必用短路径**（如 `C:\PowerBI助手`）。
> 本包内含 `anthropic` 等库有很长的文件名，配合长目录会超过 Windows 260 字符上限，导致解压/安装时
> 报 `No such file or directory`。**发给同事时一定要提醒他们解压到 `C:\` 下的短目录。**

### 同事侧
解压到**短路径**（如 `C:\PowerBI助手`）→ 双击 `packaging\启动.bat` → 浏览器自动开 → ⚙ 填自己的 Key → 连接 Power BI。

> 提示：本仓库已用上述流程产出并**实测验证过**一个完整零前提包
> （`C:\pbibuild\PowerBI助手.zip`，约 161MB，Python 3.12.13 + 全部运行期依赖）。
> 已验证：解压到全新短路径后，`启动.bat` 能用包内 `runtime\python.exe` 独立起服务。
> 未验证（固有限制）：真正无网/无 Python/无 Power BI 的全新电脑——首发前请在这样一台机器上再走一遍自检。

## 方案一+：打成安装包 exe（Inno Setup，最"像装软件"）

在方案一的便携 bundle 基础上，用 Inno Setup 打成一个 `setup.exe`：双击安装、自动建开始菜单/桌面快捷方式（指向 `启动.vbs` = 应用模式窗口）、带卸载程序。**免管理员，装到 `%LOCALAPPDATA%\PowerBI助手`。**

脚本：[installer.iss](installer.iss)（已随仓库；必须保存为 **UTF-8 with BOM**，否则中文串编译乱码）。

步骤：
1. 装 Inno Setup（一次）：`winget install --id JRSoftware.InnoSetup -e`（ISCC 在 `%LOCALAPPDATA%\Programs\Inno Setup 6\ISCC.exe`）。
2. 准备好方案一的 bundle 目录（含 `runtime\` 等）；放一个 `icon.ico` 在其根目录（可用包内 pillow 生成）。
3. 编译：`"<...>\ISCC.exe" /DSrcDir="<bundle 目录>" installer.iss` → 产出 `C:\pbibuild\PowerBI助手_安装.exe`。

> 本仓库已用此流程产出并**实测验证**过 `C:\pbibuild\PowerBI助手_安装.exe`（约 96MB）：静默安装可正确铺开全部文件 + 卸载程序，开始菜单快捷方式指向 `启动.vbs`，卸载能干净移除。安装向导是英文（应用本身中文）。

### 同事侧（安装包）
双击 `PowerBI助手_安装.exe` → 一路 Next/Install → 从开始菜单或桌面「Power BI 助手」启动（应用模式窗口）→ ⚙ 填自己的 Key → 连接 Power BI。卸载走「卸载 Power BI 助手」或控制面板。

## 方案二：轻量包（同事自行装一次 Python 3.12）

不打包 `runtime\`，包小很多。`启动.bat` 检测到没有 `runtime\` 且没有 `.venv\` 时，会用系统 `python` **首次自动创建 `.venv` 并联网安装依赖**（仅第一次，几分钟）。

适合：同事愿意装一次 [Python 3.12](https://www.python.org/downloads/)（安装时务必勾选 *Add Python to PATH*）。

组装目录同方案一，但**去掉 `runtime\`**。

## 发版前自检（务必在一台"干净"机器或新用户上验证）
- [ ] 双击 `启动.bat` 能起来、浏览器自动打开。
- [ ] ⚙ 填 Key 后能正常对话生成。
- [ ] 打开一个 Power BI 报表后，左侧能连上（绿灯）、能实跑验证。
- [ ] 开写入开关能写回度量值（首次可能需联网下载 TOM）。
- [ ] 路径里带空格/中文时也能跑（`启动.bat` 已用引号处理，但建议测一下）。

## 升级分发
代码有更新时：方案一只需替换 `powerbi_ai_assistant\` 目录重新打包（`runtime\` 不变，除非依赖版本变了）；依赖变了则重跑第 2 步。
