On this project the user repeatedly steered toward: **confirm details before writing code** ("不要急着开发，我需要确认好所有细节" / "先生成文档就行"), and toward **rigorous, evidence-based evaluation** rather than assertions (asked to run skill benchmarks, then to raise difficulty with harder cases).

**Why:** they're building a real product and want decisions grounded in evidence, not optimistic claims. They responded well to honest findings (e.g. "the baseline model is already strong; the skill's lift is small / mostly discipline") and to surfacing a shared latent bug rather than glossing it.

**How to apply:** plan/scope and confirm before scaffolding or generating large code. Give honest, calibrated assessments — state when something is a tie, a limitation, or unverified; don't oversell. When evaluating quality, prefer real execution/benchmarks over assertion. Offer clear A/B/C next-step choices. Part of [[powerbi-ai-assistant-project]].
